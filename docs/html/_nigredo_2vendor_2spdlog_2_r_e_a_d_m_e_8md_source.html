7 ## Install
8 #### Just copy the headers:
9 
10 * Copy the source [folder](https://github.com/gabime/spdlog/tree/v1.x/include/spdlog) to your build tree and use a C++11 compiler.
11 
12 #### Or use your favorite package manager:
13 
14 * Ubuntu: `apt-get install libspdlog-dev`
15 * Homebrew: `brew install spdlog`
16 * FreeBSD: `cd /usr/ports/devel/spdlog/ && make install clean`
17 * Fedora: `yum install spdlog`
18 * Gentoo: `emerge dev-libs/spdlog`
19 * Arch Linux: `yaourt -S spdlog-git`
20 * vcpkg: `vcpkg install spdlog`

29 ## Features
30 * Very fast (see [benchmarks](#benchmarks) below).
31 * Headers only, just copy and use.
32 * Feature rich formatting, using the excellent [fmt](https://github.com/fmtlib/fmt) library.
33 * Fast asynchronous mode (optional)
34 * [Custom](https://github.com/gabime/spdlog/wiki/3.-Custom-formatting) formatting.
35 * Multi/Single threaded loggers.
36 * Various log targets:
37  * Rotating log files.
38  * Daily log files.
39  * Console logging (colors supported).
40  * syslog.
41  * Windows debugger (```OutputDebugString(..)```)
42  * Easily extendable with custom log targets (just implement a single function in the [sink](include/spdlog/sinks/sink.h) interface).
43 * Severity based filtering - threshold levels can be modified in runtime as well as in compile time.
44 * Binary data logging.

78 ## Usage samples
79 
80 ```c++
81 #include "spdlog/spdlog.h"
82 #include "spdlog/sinks/stdout_color_sinks.h"
83 void stdout_example()
84 {
85  // create color multi threaded logger
86  auto console = spdlog::stdout_color_mt("console");
87  console->info("Welcome to spdlog!");
88  console->error("Some error message with arg: {}", 1);
89 
90  auto err_logger = spdlog::stderr_color_mt("stderr");
91  err_logger->error("Some error message");
92 
93  // Formatting examples
94  console->warn("Easy padding in numbers like {:08d}", 12);
95  console->critical("Support for int: {0:d}; hex: {0:x}; oct: {0:o}; bin: {0:b}", 42);
96  console->info("Support for floats {:03.2f}", 1.23456);
97  console->info("Positional args are {1} {0}..", "too", "supported");
98  console->info("{:<30}", "left aligned");
99 
100  spdlog::get("console")->info("loggers can be retrieved from a global registry using the spdlog::get(logger_name)");
101 
102  // Runtime log levels
103  spdlog::set_level(spdlog::level::info); // Set global log level to info
104  console->debug("This message should not be displayed!");
105  console->set_level(spdlog::level::trace); // Set specific logger's log level
106  console->debug("This message should be displayed..");
107 
108  // Customize msg format for all loggers
109  spdlog::set_pattern("[%H:%M:%S %z] [%n] [%^---%L---%$] [thread %t] %v");
110  console->info("This an info message with custom format");
111 
112  // Compile time log levels
113  // define SPDLOG_DEBUG_ON or SPDLOG_TRACE_ON
114  SPDLOG_TRACE(console, "Enabled only #ifdef SPDLOG_TRACE_ON..{} ,{}", 1, 3.23);
115  SPDLOG_DEBUG(console, "Enabled only #ifdef SPDLOG_DEBUG_ON.. {} ,{}", 1, 3.23);
116 }
117 ```

181 ```c++
182 // log binary data as hex.
183 // many types of std::container<char> types can be used.
184 // ranges are supported too.
185 // format flags:
186 // {:X} - print in uppercase.
187 // {:s} - don't separate each byte with space.
188 // {:p} - don't print the position on each line start.
189 // {:n} - don't split the output to lines.
190 
191 #include "spdlog/fmt/bin_to_hex.h"
192 
193 void binary_example()
194 {
195  auto console = spdlog::get("console");
196  std::array<char, 80> buf;
197  console->info("Binary example: {}", spdlog::to_hex(buf));
198  console->info("Another binary example:{:n}", spdlog::to_hex(std::begin(buf), std::begin(buf) + 10));
199  // more examples:
200  // logger->info("uppercase: {:X}", spdlog::to_hex(buf));
201  // logger->info("uppercase, no delimiters: {:Xs}", spdlog::to_hex(buf));
202  // logger->info("uppercase, no delimiters, no position info: {:Xsp}", spdlog::to_hex(buf));
203 }

208 #### Logger with multi sinks - each with different format and log level
209 ```c++
210 
211 // create logger with 2 targets with different log levels and formats.
212 // the console will show only warnings or errors, while the file will log all.
213 void multi_sink_example()
214 {
215  auto console_sink = std::make_shared<spdlog::sinks::stdout_color_sink_mt>();
216  console_sink->set_level(spdlog::level::warn);
217  console_sink->set_pattern("[multi_sink_example] [%^%l%$] %v");
218 
219  auto file_sink = std::make_shared<spdlog::sinks::basic_file_sink_mt>("logs/multisink.txt", true);
220  file_sink->set_level(spdlog::level::trace);
221 
222  spdlog::logger logger("multi_sink", {console_sink, file_sink});
223  logger.set_level(spdlog::level::debug);
224  logger.warn("this should appear in both console and file");
225  logger.info("this message should not appear in the console, only in the file");
226 }
227 ```
